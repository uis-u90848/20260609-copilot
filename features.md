# ポモドーロタイマーアプリ 実装機能一覧

[architecture.md](architecture.md) と UI モックを踏まえて実装が必要な機能を洗い出す。
優先度は **MVP**（最小限）／**拡張**（任意）で区分する。

## フロントエンド（ブラウザ側）

### タイマーコア（MVP）
- [ ] 作業時間のカウントダウン（デフォルト25分、`MM:SS` 表示）
- [ ] 開始 / 一時停止 / リセット操作
- [ ] タイマーの状態管理（Idle / Running / Paused / Completed の状態機械）
- [ ] 開始時刻＋設定時間からの残り時間算出（タブ非アクティブでもズレない方式）
- [ ] 0秒到達時の完了処理

### 作業／休憩のサイクル（MVP）
- [ ] 作業（25分）→ 短い休憩（5分）の自動切替
- [ ] 4ポモドーロごとに長い休憩（15分）へ切替
- [ ] 現在が「作業中／休憩中」かの表示

### UI 表示（MVP）
- [ ] 円形プログレス（残り時間に応じてリング更新：SVG または `conic-gradient`）
- [ ] 残り時間の数値表示
- [ ] 今日の進捗表示（完了ポモドーロ数・合計集中時間）
- [ ] モック準拠のレイアウト・スタイル（カード型 UI）

### 通知・フィードバック（拡張）
- [ ] セッション完了時の音（アラーム）
- [ ] ブラウザ通知（Notification API）
- [ ] タブのタイトル更新（残り時間をタブに表示）

### 設定（拡張）
- [ ] 作業／休憩時間のカスタマイズ
- [ ] 設定の `localStorage` 保存

## バックエンド（Flask）

### ページ配信（MVP）
- [ ] `GET /` でメイン画面（`index.html`）を返す
- [ ] 静的ファイル（CSS / JS）の配信

### 進捗API（MVP）
- [ ] `GET /api/progress`：今日の進捗（完了数・合計集中分）を返す
- [ ] `POST /api/sessions`：1ポモドーロ完了を記録（種類・長さを送信）

### サービス層・データ層（MVP）
- [ ] 今日の進捗を集計するロジック（`services.py`）
- [ ] 入力バリデーション（種類・長さの検証、不正値拒否）（`validators.py`）
- [ ] SQLite への記録・取得（`repository.py`）
- [ ] セッション記録テーブルのスキーマ・初期化
- [ ] `create_app()` アプリケーションファクトリ

### 拡張（任意）
- [ ] 日別・週別の集計や履歴取得API
- [ ] ユーザー認証（案C相当）

## 横断的・基盤

### テスト（テスト容易性重視のため重要）
- [ ] サービス層の単体テスト（`pytest`）
- [ ] バリデーションの単体テスト
- [ ] 時刻依存テスト（`freezegun` で日付境界）
- [ ] API 結合テスト（Flask `test_client()`）
- [ ] タイマーロジックのテスト（`vitest`/`jest`、`now` 注入）

### プロジェクト基盤
- [ ] `requirements.txt` / `requirements-dev.txt`
- [ ] エラーハンドリング（API のエラーレスポンス）

## MVP として最初に実装すべき範囲

まず動くものを作るための最小セット（実装推奨順）。

1. Flask の `GET /` 配信 + `create_app()`
2. `index.html` + `style.css`（モック準拠）
3. `timer.js`（カウントダウン・開始/リセット・状態機械）
4. 円形プログレス表示
5. `GET /api/progress` / `POST /api/sessions` + サービス層・SQLite
6. 作業→休憩の自動切替

通知音・設定カスタマイズ・履歴・認証は拡張フェーズに回す。
